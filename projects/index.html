Projects

I build tools, SDKs, and systems that make engineering simpler, smarter, and more scalable — from developer utilities to intelligent data processing engines. Below are a few selected open-source projects I’ve authored and maintain.

FinDocParserPro

Project Repo

A modular, extensible financial document parsing engine designed for extracting structured data from invoices, receipts, and credit notes. It combines OCR, regex-based field extraction, and clean architecture principles to provide consistent, testable, and scalable document ingestion.

Highlights

  • CQRS without MediatR — lean command/query separation for clarity and performance
  • FluentResults for predictable, exception-free flow control
  • Minimal API design — clean endpoints like:
app.MapPost("/invoices", async (CreateInvoiceCommand cmd, CreateInvoiceHandler h) => await h.Handle(cmd));
  • Layered domain enforcing separation of concerns (Domain, Application, Infrastructure, API).
  • Aimed at developers building ERP, accounting, or auditing systems that need OCR-driven financial intelligence.

go-bloomfilter

A probabilistic data structure implementation for ultra-fast membership testing. Optimized for learning and efficiency — trades small false positives for extreme speed and memory savings.

Features

  • Compact []uint64 bit array (bitwise optimized)
  • Configurable hash count and false positive rate
  • Auto-tuned parameters for best memory–accuracy balance
  • MurmurHash-inspired hashing for consistent distribution

Ideal for backend engineers exploring Go internals, bitwise operations, or algorithmic optimization.

go-strict (Cognitive Complexity Linter)

A custom linter that quantifies the cognitive complexity of Go functions — helping teams write code that’s readable, maintainable, and less error-prone.

Highlights

  • Analyzes function nesting and control flow
  • Reports complexity scores inline with file and line context
  • Configurable thresholds and output formats (text, json, xml)
  • Encourages thoughtful design through complexity awareness

Example

Function main (main.go:5): 3
  IfStmt (main.go:7): 1
  ForStmt (main.go:12): 1

Aimed at developers practising Clean Code and software craftsmanship in Go.

Africa’s Talking Go SDK

A full-featured Go SDK for Africa’s Talking APIs — simplifying integrations for SMS, Voice, USSD, Airtime, and Payments.

Capabilities

  • Send and receive SMS
  • Make and manage voice calls
  • Process payments via mobile money or card
  • Build USSD menus with minimal setup

Example Usage

client := africastalking.NewClient("USERNAME", "API_KEY")
res, err := client.SMS.Send(africastalking.SmsSender{
  Recipients: []string{"+254712345678"},
  Message:    "Hello from Africa's Talking!",
})
